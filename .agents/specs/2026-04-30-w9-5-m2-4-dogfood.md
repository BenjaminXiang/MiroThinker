# W9-5: M2.4 主页论文采集 dogfood R3 验收

## 1. Goal

M2.4 (`paper/homepage_ingest.py` + `scripts/run_homepage_paper_ingest.py`) 代码 + 单测齐全，但 R3 acceptance gate (10 profs × ≥ 15 papers/prof 真实挂链) 没有实测验证证据。`docs/solutions/integration-issues/homepage-paper-ingest-dogfood-template-2026-04-22.md` 仍标 `status: template-pending-execution`。

### 5.2 Step 1: Dry-run 10 教授

```bash
cd apps/miroflow-agent
unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY all_proxy ALL_PROXY

DATABASE_URL=$DATABASE_URL uv run python scripts/run_homepage_paper_ingest.py \
  --dry-run \
  --limit 10 2>&1 | tee /tmp/w9-5-dry-run.log
```

**Record 到归档文件**：
- 启动时间、duration
- profs_processed / profs_skipped / papers_linked_total (would-be) / full_text_fetched_total (would-be) / pipeline_issues_filed (would-be)
- per-prof JSONL（从 stdout 抓出）
- Observations: 哪些 profs 0 publication（标 JS 渲染 follow-up）；resolution hit rate by source

### 5.3 Step 2: Wet-run 5 教授真实写入

挑 dry-run 结果好的 5 个 profs（high paper count, clean resolution）。如果不好挑则从 limit 5 默认。

**Before SQL** (写到归档):
```sql
SELECT count(*) FROM professor_paper_link WHERE evidence_source_type='personal_homepage';
SELECT count(*), source FROM paper_full_text GROUP BY source;
SELECT count(*), match_source FROM paper_title_resolution_cache GROUP BY match_source;
```

**Run**:
```bash
DATABASE_URL=$DATABASE_URL uv run python scripts/run_homepage_paper_ingest.py \
  --limit 5 2>&1 | tee /tmp/w9-5-wet-run.log
```

记录 run_id 输出。

**After SQL** (写到归档):
```sql
SELECT count(*) FROM professor_paper_link WHERE evidence_source_type='personal_homepage';
SELECT count(*), source FROM paper_full_text GROUP BY source;
SELECT count(*), match_source FROM paper_title_resolution_cache GROUP BY match_source;
SELECT issue_type, count(*) FROM pipeline_issue WHERE run_id = '<this-run-id>' GROUP BY issue_type;
```

### 5.4 R3 gate 校验

检查 `papers_linked_total` ≥ 10 profs × 15 papers = 150？这是 dry-run 数字。或者按 wet-run 5 profs × 15 papers = 75？

**判定**：
- 至少 R3 模板要求 10 profs × ≥ 15 papers/prof。dry-run 10 教授中 ≥ 7 个有 ≥ 15 papers 才算"基本满足"
- 实际数字写到归档 §"R3 gate 评估"

如未达 R3：写明哪些 profs 不达标 + 推测原因（JS 渲染 / 主页结构异常 / OpenAlex 匹配差），不强制阻塞 W9-5 完成（运维已知问题，留作 W12-3 follow-up）。

## 7. Invariants

1. 归档文件 `status: complete`（区别于 template 的 `template-pending-execution`）
2. 文件名含日期；同日重复跑后缀 `-v2` / `-v3`
3. 所有 SQL 输出粘贴**真实数字**（不要伪造）
4. run_id 必须真实记录（V007 起 pipeline_run 表保有）
5. DSN 写入时只暴露 host + db name，不带密码
6. 不修改 M2 代码（仅运行 + 记录）
7. dry-run 必须先于 wet-run（避免直接污染 canonical）
8. 实测数字归档完毕前**不**关闭本 W9-5 任务

## 8. Edge cases / 失败模式

| 场景 | 处理 |
|---|---|
| Dry-run 中所有 prof 0 publication | 检查 homepage URL 是否大批 JS 渲染；如是，注明 + 标 W12-3 follow-up；R3 标 fail |
| OpenAlex / Serper 限速 | 重试（M2.4 已有 retry）；如全部失败，注明并 R3 标 partial |
| Wet-run 中途崩 | 用 `professor_paper_link` 的 run_id 找已写入行；归档前文件 dry-run + 已部分写入的 wet-run；标 partial |
| Pre-flight V011 未应用 | stop；先 alembic upgrade，不要硬跑 |
| 教授库为空（institution filter 太严） | 用更宽 filter（如 `--institution "南方科技大学"`）；归档说明用了什么 filter |
