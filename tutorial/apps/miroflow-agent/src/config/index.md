# config 模块概览

## 模块简介

`config` 模块是 MiroThinker 智能体框架的**配置中心**，负责管理所有外部服务的 API 密钥、MCP 工具服务器的启动参数，以及环境信息的收集。它是整个系统启动时最先被加载的模块之一。

## 文件清单

| 文件 | 说明 |
|------|------|
| `settings.py` | 核心配置文件，包含环境变量加载、MCP 服务器参数生成、子智能体暴露、环境信息收集 |
| `__init__.py` | 包初始化文件，导出 `create_mcp_server_parameters`、`expose_sub_agents_as_tools`、`get_env_info` |

## 设计理念

该模块遵循**单一职责原则**：所有与外部配置相关的逻辑集中在此，其他模块通过导入本模块来获取所需的配置参数，而不是各自独立读取环境变量。这样做的好处是：

1. API 密钥只在一个地方加载，便于审计和安全管理
2. MCP 服务器的启动参数集中管理，添加新工具只需修改一个文件
3. 环境信息收集统一化，避免重复代码
