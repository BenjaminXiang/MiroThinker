# io 模块概览

## 模块简介

`io` 模块负责 MiroThinker 智能体的**输入处理和输出格式化**。它是系统与外界数据交互的"翻译层"：将各种格式的用户输入（PDF、图片、音频、Excel 等）转换为 LLM 能理解的文本，同时将 LLM 的原始输出格式化为结构化的最终答案。

## 文件清单

| 文件 | 说明 |
|------|------|
| `input_handler.py` | 输入处理器，支持 20+ 种文件格式的解析和媒体内容提取 |
| `output_formatter.py` | 输出格式化器，处理工具结果截断、`\boxed{}` 答案提取、Token 统计 |
| `__init__.py` | 包初始化文件，导出 `process_input` 和 `OutputFormatter` |

## 设计理念

该模块的核心设计思想是**格式透明化**：无论用户提交的是什么格式的文件，经过 `input_handler` 处理后，LLM 看到的都是统一的 Markdown 文本。这使得核心推理逻辑不需要关心输入的原始格式。

对于输出，`output_formatter` 确保了从 LLM 的自由文本中可靠地提取出结构化的答案（通过 `\boxed{}` 语法），并提供了工具结果的截断机制防止上下文溢出。
