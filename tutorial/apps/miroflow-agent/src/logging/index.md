# logging 模块概览

## 模块简介

`logging` 模块是 MiroThinker 的**任务执行追踪系统**。它提供了结构化的日志记录功能，将智能体的每一步操作（LLM 调用、工具执行、子智能体会话等）记录到内存中，并在任务结束后持久化为 JSON 文件。同时，它还提供了基准测试结果的汇总统计功能。

## 文件清单

| 文件 | 说明 |
|------|------|
| `task_logger.py` | 核心日志系统，包含 TaskLog、StepLog 等数据类和彩色控制台输出 |
| `summary_time_cost.py` | 基准测试结果汇总统计，生成时间和成本的总结报告 |
| `__init__.py` | 包初始化，导出 `TaskLog`、`StepLog`、`LLMCallLog`、`ToolCallLog`、`bootstrap_logger`、`get_utc_plus_8_time` |

## 设计理念

该模块的核心设计是**结构化日志**：不是简单的文本行日志，而是将每个事件记录为带有时间戳、级别、步骤名称和元数据的结构化对象。这使得日志既可以在控制台实时查看（带颜色标记），也可以在事后以 JSON 格式进行程序化分析。

`TaskLog` 作为单一数据容器贯穿整个任务生命周期，所有组件都通过它的 `log_step()` 方法记录事件，确保了日志的统一性和完整性。
