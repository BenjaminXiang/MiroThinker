# Task: Add structured JSON output mode to the current runtime

## Description

Extend the current `miroflow-agent` runtime so data-agent tasks can emit validated JSON objects instead of benchmark-only `\boxed{}` answers. This task owns the cross-cutting runtime changes in `input_handler`, `prompt_utils`, `output_formatter`, `answer_generator`, `orchestrator`, and `pipeline`.

## Acceptance Criteria

- [ ] `process_input` can skip the boxed-answer instruction for structured tasks.
- [ ] `generate_agent_summarize_prompt` supports a JSON output mode with an explicit schema payload.
- [ ] `OutputFormatter` can extract canonical JSON payloads from the final assistant message.
- [ ] The default runtime behavior for existing benchmark tasks remains boxed mode.
- [ ] Structured-output extraction is deterministic and test-covered.

## Technical Details

- Keep boxed mode as the default in `apps/miroflow-agent/conf/agent/default.yaml`.
- Thread `output_mode` and `final_output_schema` through the existing runtime rather than branching into a separate pipeline.
- Use the existing JSON repair helper in `apps/miroflow-agent/src/utils/parsing_utils.py` instead of adding another ad-hoc parser.
- Do not touch domain-specific code in this issue.
