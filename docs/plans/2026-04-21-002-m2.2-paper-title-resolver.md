# M2.2 — Paper Title Resolver

## Overview

Given a clean paper title plus optional author and year hints (output of M2.1's `HomepagePublication`), return a `ResolvedPaper` containing enough structured metadata to upsert the paper into our canonical store and — in M2.3 — fetch its full text. If nothing is confidently found, return `None`.

This closes the gap between "we know the homepage-authoritative title" (M2.1) and "we have a row in `paper` with DOI/arxiv/abstract" (M2.4). It is the second brick of the Homepage-Authoritative pipeline that 001 §4.4 identified as the single highest-leverage fix for the 47% paper-reject rate.

No DB writes. No PDF downloads. No feature-flag gating. Just HTTP cascade + similarity scoring + best-effort metadata normalization.

## Problem Frame

After M2.1 parses a professor's homepage, we have a list of `HomepagePublication` records — each is essentially `{title, maybe authors, maybe year, maybe DOI/arxiv anchor}`. Downstream we need the canonical record: a real DOI or arxiv ID, full author list, venue, year, and abstract. Three good sources exist:

- **OpenAlex** — free, no auth, best title-search semantics (`/works?search="..."`), and it returns `abstract_inverted_index` you can rebuild into prose. Should resolve ~80% of real papers.
- **arxiv** — free, rate-limited, title-search via `ti:"..."` on the Atom API. Catches CS/math preprints OpenAlex sometimes misses.
- **Serper** — Google search API via the existing `WebSearchProvider`. Last-resort fallback for papers neither OpenAlex nor arxiv know about (e.g., recent Chinese conferences, industry whitepapers).

Without a title resolver, M2.4 either (a) skips the homepage-parsed titles (losing the whole point of M2.1), or (b) hits production databases with ungrounded "title-only" paper records (ops nightmare). So this milestone is a must-have between M2.1 and M2.4.

## Requirements Trace

- **R1** (from 003 §M2.2): Signature `resolve_paper_by_title(clean_title: str, *, author_hint: str | None = None, year_hint: int | None = None, web_search: WebSearchProvider | None = None) -> ResolvedPaper | None`.
- **R2** (from 003 §M2.2): `ResolvedPaper` fields: `title`, `doi`, `openalex_id`, `arxiv_id`, `abstract`, `pdf_url`, `authors`, `year`, `venue`, `match_confidence (0..1)`, `match_source ("openalex" | "arxiv" | "web_search")`.
- **R3** (from 003 §M2.2): Cascade OpenAlex → arxiv → web_search. Stop at first result with `match_confidence ≥ 0.85`.

- **R5** (from 003 §M2.2): Respect rate limits — OpenAlex 10/s, arxiv 1 req / 3s. Web search cadence is bounded by Serper's provider.
- **R6** (derived): Pure HTTP. No DB writes, no filesystem side effects, no PDF downloads.
- **R7** (derived): Hermetic tests. No live network calls during `uv run pytest`. All three external APIs mocked.
- **R8** (derived from feature brief): Expose a cache injection point (Protocol) without implementing or depending on the actual V011 cache table. Default is no cache. M2.4 wires the real cache later.
- **R9** (derived from feature brief): Reconstruct OpenAlex's `abstract_inverted_index` into a plain abstract string when available. Free metadata, needed by M2.3 and M6.

## Key Technical Decisions

- **Single new module: `apps/miroflow-agent/src/data_agents/paper/title_resolver.py`**. Co-locate the three source helpers + orchestrator + dataclass in one file. ~400-500 lines total estimated. Small enough to comprehend, big enough to avoid fragmentation. Adding three separate modules (openalex_title_search.py, arxiv_client.py, title_resolver.py) fragments the cascade semantics without buying testability — the cascade IS the interesting logic.
- **Sync, not async.** Same reasoning as M0.1 RerankerClient: first callers (M2.4 backfill script, future chat path) are sync. Adding async later is cheap if a concurrent-resolver use case appears.
- **httpx with `trust_env=False` baked in.** Match `RerankerClient` shape. Accept optional `http_client: httpx.Client | None = None` for caller-owned clients (supports test injection and shared-pool reuse).
- **Jaccard on normalized tokens for similarity.** Threshold `0.85` per 003. Normalization: `clean_paper_title` → lowercase → strip punctuation → split on whitespace → token set. This is distinct from M2.1's dedup normalizer (which collapses to exact-equality key).
- **Confidence = base Jaccard, with additive boosts capped at 1.0:**
  - `+0.05` if `year_hint` is provided and matches the source's year exactly
  - `+0.05` if `author_hint` is provided and any author surname appears in source authors (case-insensitive)
- **Cascade short-circuits at ≥ 0.85.** If OpenAlex returns a match of 0.90, skip arxiv and Serper. If it returns 0.82, fall through. This means one clean title-match halts the cascade early, saving rate-limit budget.
- **Web-search source is opt-in.** `web_search: WebSearchProvider | None = None` default. If None, cascade stops at arxiv. M2.4 orchestrator can inject a provider when Serper quota is available.
- **Cache hook is a Protocol, not a class.** `TitleResolutionCache` protocol with two methods (`get(key) -> ResolvedPaper | None`, `set(key, value)`). Default None. M2.4 wires a Postgres-backed implementation; tests use a dict-backed fake.
- **Cache key = `sha1(hex)(_normalize_title_for_match(clean_title))`.** Case + punctuation + whitespace insensitive. Year/author hint are NOT part of the key — they're ranking signals, not identity signals.
- **Rate-limit gate is module-level singleton with monotonic-time reservations.** Simple, no deps. Two gates: `_OPENALEX_GATE` (min 0.1s between calls) and `_ARXIV_GATE` (min 3.0s between calls). Thread-safe enough via lock; not async-safe (sync-only).
- **Abstract reconstruction is best-effort.** If the inverted index is empty, missing, or has positions > 5000, return None. Don't try to reconstruct partials.
- **Never raise on external-API flakiness.** HTTP 429 → log warning, treat as "source returned no match", continue cascade. HTTP 5xx → same. XML parse error → same. Only `TypeError`/invalid args raise.
- **Reuse `WebSearchProvider` — do NOT port it to httpx in this milestone.** M0.5 owns that migration; this milestone accepts the current `requests`-based provider as-is.

## Open Questions

### Resolved During Planning

- **Q: New module or split into three?** → Single `title_resolver.py`. See Key Technical Decisions.
- **Q: Sync or async?** → Sync.
- **Q: Jaccard, Levenshtein, or embedding cosine?** → Jaccard. Cheap, deterministic, easy to test, mirrors the "token-set similarity" intuition for paper titles.
- **Q: Confidence threshold?** → 0.85 per 003 §M2.2.
- **Q: Include OpenAlex abstract reconstruction?** → Yes. It's free metadata on the same response, M2.3 and M6 need it.
- **Q: Cache implementation here or in M2.4?** → Protocol hook here, implementation in M2.4.
- **Q: Retry on 5xx?** → No. Keep simple; cascade naturally falls through to next source on failure.
- **Q: What if Serper returns a result with high Jaccard but it's a Wikipedia page / PDF preview / SSRN draft?** → Filter organic results to those whose `link` domain matches known scholarly hosts (arxiv.org, doi.org, *.acm.org, semanticscholar.org, researchgate.net, nature.com, openreview.net, pubmed). Discard others. Matches the "scholarly output" signal from `docs/solutions/best-practices/discipline-aware-professor-quality-gate-2026-04-14.md` in spirit.
- **Q: Include year in cache key?** → No. Title identity is the paper; year is a ranking boost.
- **Q: What Jaccard value does a near-identical title yield?** Answer example: "Deep Learning for Images" vs "Deep Learning for Image" — 3/4 intersect, 4 union = 0.75 below threshold without hints. With year hint match → 0.80 still below. With author hint match → 0.85 exactly at threshold. Keep this in mind for fuzzy-title tests.

### Deferred to Implementation

- **Exact token normalization for matching.** Start conservative (lowercase, strip `.,:;!?()[]{}"`), extend as test failures on real titles surface edge cases. Don't try to get it right without fixtures.
- **How aggressively to strip title decorations in OpenAlex response.** OpenAlex sometimes returns titles with trailing "<i>...</i>" HTML. `clean_paper_title` already handles this. Implementer confirms at GREEN time.
- **Exact mapping of arxiv version suffix to `arxiv_id`.** arxiv returns IDs like `2304.12345v3`. Plan says `arxiv_id` is the bare form (`2304.12345`). Strip at implementation.
- **Exact Serper payload shape to parse.** Implementer reads the fixture responses at RED time and pins the exact keys (organic[*].title, link, snippet, etc.).
- **Whether to split the `authors` field at `,` / `and`.** OpenAlex returns `authorships[*].author.display_name` as a list; arxiv gives a list. Both produce `list[str]`. Serper gives no structured authors; implementer decides whether to leave authors=[] or try to parse from snippet. Default: leave empty; let downstream M2.4 enrich from the resolved DOI.

## High-Level Technical Design

> *This illustrates the intended approach and is directional guidance for review, not implementation specification. The implementing agent should treat it as context, not code to reproduce.*

```
resolve_paper_by_title(clean_title, *, author_hint, year_hint, web_search, http_client, cache)
│
├─ key = sha1(_normalize_title_for_match(clean_title))
├─ if cache: cached = cache.get(key); if cached is not None: return cached
│
├─ 1. OpenAlex
│   │   _OPENALEX_GATE.wait()
│   │   GET https://api.openalex.org/works?search="<clean_title>"&per-page=5
│   │   &select=id,doi,title,publication_year,host_venue,authorships,abstract_inverted_index
│   │   for each work in response.results:
│   │     conf = _title_jaccard(clean_title, work.title)
│   │     conf += year_boost(year_hint, work.publication_year)
│   │     conf += author_boost(author_hint, work.authorships)
│   │     if conf > best.confidence: best = (work, conf, "openalex")
│   │   if best.confidence >= 0.85:
│   │     result = ResolvedPaper(source="openalex", abstract=reconstruct_inverted(...), ...)
│   │     cache?.set(key, result); return result
│
├─ 2. arxiv  (only if OpenAlex didn't meet threshold)
│   │   _ARXIV_GATE.wait()
│   │   GET https://export.arxiv.org/api/query?search_query=ti:"<title>"&max_results=5
│   │   for each <entry> in atom feed:
│   │     (same similarity scoring; source="arxiv")
│   │   if best >= 0.85: cache?.set; return
│
├─ 3. Serper (only if web_search is not None AND both above missed)
│   │   response = web_search.search(clean_title)
│   │   organic = filter(result => result.link.domain in _SCHOLARLY_DOMAINS)
│   │   for each organic:
│   │     build candidate title from organic.title (strip "... - arXiv" suffix etc.)
│   │     (similarity scoring; source="web_search")
│   │   if best >= 0.85: cache?.set; return
│
└─ return None   (no source produced ≥ 0.85)
```

This design keeps the cascade obvious on first read. Each stage is independent — failure of one is observable and advances to the next.

## Implementation Units

- [ ] **Unit 1: `ResolvedPaper` dataclass + title-similarity helpers**

**Goal:** Define the output contract and the pure helpers that score matches and reconstruct abstracts.

**Requirements:** R2, R9

**Dependencies:** None.

**Files:**
- Create: `apps/miroflow-agent/src/data_agents/paper/title_resolver.py` (dataclass + `_normalize_title_for_match`, `_tokenize_for_match`, `_title_jaccard`, `_reconstruct_abstract_from_inverted_index`, `_title_cache_key`, `TitleResolutionCache` Protocol — all module-private helpers except `ResolvedPaper` and `TitleResolutionCache`).
- Test: `apps/miroflow-agent/tests/data_agents/paper/test_title_resolver.py` (new file).

**Approach:**
- `ResolvedPaper` frozen slotted dataclass with R2 fields + `authors: tuple[str, ...]` for hashability (not `list`).
- `_normalize_title_for_match(text)`: run through `clean_paper_title` first, then casefold, strip punctuation via regex, collapse whitespace. Return normalized string.
- `_tokenize_for_match(text)`: `_normalize_title_for_match` + split on whitespace. Return `frozenset[str]`.
- `_title_jaccard(a, b)`: set-intersection / union over token frozensets. Return 0.0 if either is empty.
- `_reconstruct_abstract_from_inverted_index(inverted)`: given `{word: [positions]}`, produce `"word1 word2 word3 ..."` by placing each word at each of its positions, iterating in position order. Cap max position at 5000 (reject absurd inputs). Return None if inverted is empty/None/malformed.
- `_title_cache_key(clean_title)`: sha1 hex of `_normalize_title_for_match(clean_title)`.
- `TitleResolutionCache(Protocol)` with two methods: `get(key: str) -> ResolvedPaper | None` and `set(key: str, value: ResolvedPaper) -> None`.

**Execution note:** Test-first. Write the helper tests first, see them fail (ModuleNotFoundError is acceptable RED), implement.

**Patterns to follow:**
- Frozen slotted dataclass: M0.1 `providers/rerank.py::RerankResult` or `paper/canonical_writer.py::PaperUpsertReport`.
- Title cleaning delegation: `professor/homepage_publications.py` reuses `paper/title_cleaner.clean_paper_title` as a base; mirror that.
- Protocol idiom: `typing.Protocol` with `runtime_checkable` optional (not required for the cache since tests use a concrete fake).

**Test scenarios:**
- Happy path — `ResolvedPaper` dataclass construction with all fields populated; `authors` is a tuple; frozen mutation raises.
- Happy path — `_title_jaccard("Deep Learning for Images", "Deep Learning for Images")` == 1.0.
- Happy path — `_title_jaccard("Deep Learning for Images", "DEEP  LEARNING for Images!")` == 1.0 (case + punctuation + whitespace normalized).
- Happy path — `_title_jaccard("Deep Learning for Images", "Deep Learning for Image")` reflects 3/4 intersect over 5 union = 0.6 or similar — pin exact value in test once implementer's tokenization is known; accept any value in [0.5, 0.75].
- Edge case — `_title_jaccard("", "anything")` == 0.0 (empty token set short-circuits).
- Edge case — `_title_jaccard("Hello", "")` == 0.0.
- Edge case — `_title_jaccard("完全不同的标题", "Totally Different Title")` == 0.0 (no token overlap).

- [ ] **Unit 4: Orchestrator `resolve_paper_by_title` + cache hook + web search**

**Goal:** Wire the three sources into a cascade with short-circuit, cache injection, and optional Serper fallback.

**Requirements:** R1, R3, R4, R8

**Dependencies:** Units 1, 2, 3.

**Files:**
- Modify: `apps/miroflow-agent/src/data_agents/paper/title_resolver.py` (add `resolve_paper_by_title`, `_search_web_by_title`, `_web_hit_to_resolved`, `_SCHOLARLY_DOMAINS`).
- Test: same test file, appended.

**Approach:**
- `resolve_paper_by_title(clean_title, *, author_hint=None, year_hint=None, web_search=None, http_client=None, cache=None) -> ResolvedPaper | None`:
  1. Cache lookup via `_title_cache_key`. If hit → return immediately.
  2. OpenAlex search + scoring. If best confidence ≥ 0.85 → cache, return.
  3. arxiv search + scoring. If best confidence ≥ 0.85 → cache, return.
  4. If `web_search is None` → return None.
  5. Serper via `web_search.search(clean_title)`. Filter organic to `_SCHOLARLY_DOMAINS`. Score each. If best ≥ 0.85 → cache, return.
  6. Return None.
- `_SCHOLARLY_DOMAINS = {"arxiv.org", "doi.org", "acm.org", "ieee.org", "nature.com", "science.org", "sciencedirect.com", "springer.com", "openreview.net", "semanticscholar.org", "pubmed.ncbi.nlm.nih.gov", "ncbi.nlm.nih.gov", "biorxiv.org", "medrxiv.org"}`. Match on suffix (`urlparse(link).hostname.endswith(domain)`). Expand later as gaps surface.
- `_web_hit_to_resolved(organic, *, query_title, author_hint, year_hint)`: extract `title`, `link`, `snippet`. Try to derive DOI from `link` if `doi.org` host. Try to derive arxiv_id from `link` if arxiv host. `authors=()`, `venue=None`, `abstract=snippet[:500]` or None, `match_source="web_search"`. Confidence = Jaccard(query_title, title) + hints.
- Cache interactions bracket each successful return. On cache miss + cascade miss, write nothing (don't cache `None` — cheap to recompute, and downstream may re-try later with different hints).
- The function NEVER creates its own `WebSearchProvider`. Caller passes one or cascade stops at arxiv.

**Execution note:** Test-first. Build a `_FakeCache` dict-backed class in the test module; inject it to verify cache get/set calls.

**Patterns to follow:**
- Cascade short-circuit: match the spirit of `paper/hybrid.py::discover_professor_paper_candidates_from_hybrid_sources`.
- Cache-aside: same pattern as M2.4's future `paper_title_resolution_cache`.

**Test scenarios:**

*Happy path (cascade dynamics):*
- OpenAlex returns match with confidence 0.90 → `resolve_paper_by_title` returns that, does NOT invoke arxiv search or web_search (assert via `call_count` spies on mocked search helpers).
- OpenAlex returns all matches < 0.85 + arxiv returns match 0.92 → returns arxiv result, web_search not invoked.
- Both OpenAlex and arxiv miss + `web_search` provided with filtered organic matching 0.88 → returns web result.
- `author_hint` match boosts OpenAlex result from 0.82 to 0.87 → OpenAlex wins without falling through.
- `year_hint` match boosts arxiv result from 0.83 to 0.88 → arxiv wins after OpenAlex miss.

*Edge cases:*
- `clean_title = ""` → returns None immediately (no HTTP).
- All sources return empty results, no `web_search` → returns None.
- `web_search` provided but returns no organic → returns None.
- `web_search` returns organic from blog.example.com / github.com (non-scholarly) → all filtered out → returns None.
- Multiple organic results — top-scoring scholarly domain wins.

*Cache behavior:*
- Cache hit on first call → no HTTP invoked, cached value returned.
- Cache miss + OpenAlex hit → cache.set invoked with key + returned value.
- Cache miss + all miss → cache.set NOT invoked (don't cache None).

*Error paths:*
- OpenAlex helper raises unexpected RuntimeError → cascade swallows, continues to arxiv (helper contract says always returns list; but orchestrator is defensive — log + fall through).
- `web_search.search()` raises (Serper quota exceeded) → log, return None (no raise to caller).

*Contract:*
- Function signature accepts keyword-only args for all optional parameters.
- Function returns `ResolvedPaper | None`; never raises on content issues.

## Risks & Dependencies

| Risk | Mitigation |
|------|------------|
| OpenAlex changes its response shape (renames `host_venue`, etc.) | Fields wrapped in `.get()` calls, return None on missing; Unit 2 edge-case tests pin the degraded-but-not-broken behavior. |
| arxiv API rate-limits harshly despite our 3s gate (e.g., per-IP, not per-request) | Cascade falls through to Serper. Log + accept. Not this milestone's problem to retry. |
| Serper burns free quota quickly | `web_search` is opt-in. M2.4 orchestrator controls when to pass it. Default cascade stops at arxiv. |
| Jaccard misses real matches because of token differences (plurals, Chinese punctuation) | Confidence threshold 0.85 + year/author hints recover borderline cases. Real-world tuning happens in M2.4 eval loop. |
| Title-resolution cache collisions under `sha1` on normalized titles | sha1 collision probability is negligible at our scale (~10k titles). No mitigation needed; note in doc. |
| Codex over-reformats existing files (Shape 3 from memory/feedback_codex_deviations.md) | Plan explicitly lists ONLY `paper/title_resolver.py` as create target and ONLY the test file as create target. No other file should appear in the diff. Cross-validation step runs `git diff --stat` and flags any extra files. |
| Codex inlines `os.getenv` instead of using project conventions (Shape 1) | No env vars are needed for OpenAlex/arxiv (public APIs). Serper uses existing `WebSearchProvider` which handles its own key. No keys in this milestone. |
| Codex invents production shims for test-only patches (Shape 2) | Plan's test scenarios use `patch("src.data_agents.paper.title_resolver.httpx.Client")` with real-class-capture pattern documented in `docs/solutions/best-practices/httpx-module-patch-spec-mock-gotcha-2026-04-21.md`. Brief references this explicitly. |
| Abstract reconstruction produces garbage on malformed inverted index | Hard position cap 5000 + defensive None returns. Unit 1 edge cases pin the degraded behavior. |
| OpenAlex returns different titles in different cases (e.g., "Deep Learning For Images" vs "Deep Learning for Images") | `_normalize_title_for_match` casefolds and strips punctuation before tokenization. Jaccard is symmetric. |
| arxiv entry has title containing " \n " whitespace (common in arxiv's Atom output) | `clean_paper_title` collapses whitespace. Same normalization path handles it. |
