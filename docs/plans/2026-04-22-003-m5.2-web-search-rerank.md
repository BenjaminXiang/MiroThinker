# M5.2 — Web Search Rerank

## Overview

In M4 E-route, Serper organic results get `Evidence.score = 0.5` hardcoded. That's a placeholder. M5.2 replaces it with Qwen3-Reranker-8B on `(query, snippet)` pairs. Rerank score becomes the real `Evidence.score`. Top-3 by rerank wins the slice.

Single helper + 2-3 test cases.

## Requirements Trace

- **R1** (003 §M5.2): Rerank filtered organic results via `RerankerClient.rerank`.
- **R2** (003 §M5.2): Slice to top-3 after rerank.
- **R3** (derived): Preserve M4's fail-soft posture — rerank exception → use existing ordering with score=0.5 fallback.

## Scope Boundaries

**In scope:**
- New helper `_rerank_web_organics(query, scholarly_organics, reranker, top_n=3) -> list[dict]`
- E-route calls the helper between scholarly filter and Evidence wrapping
- Tests: happy + rerank-failure fallback

**Out of scope:**
- Retry logic, caching, any change outside chat.py E-route

## Key Technical Decisions

- **Helper returns list of organic dicts** (same shape Serper returned), with new `"rerank_score"` field added. E-route wraps them into Evidence with `score=rerank_score`. Avoids threading RerankResult all the way through.
- **top_n=3 hardcoded in E-route.** 3 is the plan-spec'd cap.
- **Rerank exception → return organics unchanged with `rerank_score=0.5`** — matches M4's current fallback posture.

## Implementation Units

- [ ] **Unit 1: `_rerank_web_organics` helper + wire into E-route**

**Files:**
- Modify: `apps/admin-console/backend/api/chat.py`
- Test: `apps/admin-console/tests/test_chat_retrieval.py` (extend existing file)

**Approach:**
- New module-private function `_rerank_web_organics(query: str, organics: list[dict], reranker, top_n: int = 3) -> list[dict]`:
  1. If `len(organics) == 0` → return `[]`.
  2. Docs = `[(o.get("title") or "") + " — " + (o.get("snippet") or "") for o in organics]`.
  3. Try `rerank_results = reranker.rerank(query, docs, top_n=top_n)`. On exception → log warning, return `organics[:top_n]` with `rerank_score=0.5` added.
  4. Map each `RerankResult.index` back to `organics[index]`; attach `rerank_score=result.score`.
  5. Return the mapped list (already sorted by rerank).
- E-route `_answer_knowledge_qa` (or the scholarly-filter block):
  - After `_e_route_filter_scholarly_organics` (or equivalent) produces the scholarly list, call `_rerank_web_organics(query, filtered_organics, _get_reranker_client_or_none())`. Top_n=3.
  - Wrap each returned organic as `Evidence(object_type="web", score=o["rerank_score"], snippet=o["snippet"][:500], source_url=o["link"], metadata={"title": o["title"]})`.
- `_get_reranker_client_or_none()` — safe wrapper over `deps._get_reranker_client()` returning None if init fails (to allow E-route to skip rerank and fall back to `organics[:3]` with 0.5 score).

**Test scenarios:**
- Happy — `_rerank_web_organics` given 5 organics + reranker returning top-3 by score → result has 3 items in rerank order, each with `rerank_score` populated.
- Edge — empty organics → empty list (no rerank call).
- Edge — reranker raises → return first top_n organics with `rerank_score=0.5`.
- Integration — E-route with mocked Serper returning 4 scholarly organics + mocked reranker ordering them → final Evidence list has 3 items, `.score` matches rerank output, NOT hardcoded 0.5.
