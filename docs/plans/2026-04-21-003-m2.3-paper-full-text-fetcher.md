# M2.3 ‚Äî Paper Full-Text Fetcher

## Overview

Given a `ResolvedPaper` (or its `pdf_url` + `arxiv_id`), download the PDF, extract the text of the first four pages, and split out `abstract` + `intro`. Return a `FullTextExtract` containing those fields plus `pdf_url`, `pdf_sha256` (for dedup), `source` tag, and `fetch_error`.

Pure function. No database writes. No migration. No professor linking. M2.4 wires the result into `paper_full_text`.

This is the third brick of the Homepage-Authoritative pipeline. M2.1 gave us the homepage title, M2.2 resolved it to a canonical paper with metadata (often already including an abstract from OpenAlex), and M2.3 adds the thing OpenAlex doesn't give: the introduction and additional full-text context M6 will use to augment professor `profile_summary`.

## Problem Frame

OpenAlex's `abstract_inverted_index` gives us the abstract for ~80% of papers it indexes. That's great for title resolution in M2.2 but insufficient for M6's profile reinforcement, which benefits from a paper's introduction to capture *why* this work matters to the professor ‚Äî context that abstracts often compress away. arxiv PDFs are freely available for ~60-70% of CS/ML/physics papers Shenzhen professors author, so a focused arxiv-only fetcher hits a lot of value with minimal operational risk.

Current state: no PDF fetcher in the repo. `paper/openalex.py` only reads JSON metadata; `paper/title_resolver.py` already captures `pdf_url` from arxiv's Atom response but stops there. No pdfminer integration anywhere.

Without M2.3, M6's profile-reinforcement loop has to regress to abstract-only, which we already know produces shallow `profile_summary` outputs (001 ¬ß5 documented this).

## Key Technical Decisions

- **One new module: `apps/miroflow-agent/src/data_agents/paper/full_text_fetcher.py`**. ~250 lines estimated. No sibling modules, no shared helper package. Follow M2.2's single-file pattern.
- **Sync, not async.** Matches M0.1 + M2.2. M2.4's orchestrator can wrap in a thread pool if throughput becomes a bottleneck; async-everywhere prematurely complicates testing.
- **Fallback priority** (single attempt each):
  1. If `paper.arxiv_id` is set ‚Üí construct `https://arxiv.org/pdf/<arxiv_id>.pdf`, download, extract, split. On success, `source="arxiv"`.
  2. If (1) fails OR no arxiv_id AND `paper.abstract` is non-empty ‚Üí use that as the abstract, set `intro=None`, `source="openalex"` (semantic: "came from OpenAlex inverted index via M2.2"). This path covers non-arxiv papers without pretending we fetched a PDF.
  3. If both fail ‚Üí `source="failed"`, `fetch_error="<specific reason>"`, all content fields None except `pdf_url` (retained for diagnostics).
- **`_RateLimitGate` duplicated from `paper/title_resolver.py`**. Shared-gate refactor deferred. Rationale: factoring to a shared module now requires touching M2.2's file (Codex Shape 3 territory). 20 lines of duplication is cheaper than one ruff-format round on a recent-shipped file. Note the tech debt explicitly.
- **Hash the bytes as they stream**, not after full buffer. pdfminer accepts `BytesIO`; we sha256 on a separate buffer during the response read. Avoids double-reading the bytes.
- **Call pdfminer with `maxpages=4`, default `laparams`.** 4 pages is the documented cap per 003 ¬ßM2.3. Default layout analysis is fine for academic PDFs.
- **Regex anchors for split:**
  - Abstract: `r"(?im)^\s*abstract[\s\.:\-‚Äì‚Äî]*$"` (case-insensitive, multi-line, optional punctuation after word).
  - Introduction: `r"(?im)^\s*(?:1[\s\.\)]?\s*)?introduction[\s\.:\-‚Äì‚Äî]*$"` (optional "1.", "1)" numbering prefix).
  - Subsequent section (end-of-intro marker): first `r"(?im)^\s*(?:2[\s\.\)]?\s*)?(?:related\s+work|background|methodology|methods|approach|proposed\s+method|preliminaries)[\s\.:\-‚Äì‚Äî]*$"` heading, OR hard-cap at 3000 characters.
- **Intro cap 3000 chars** per 003 ¬ßM2.3 (`intro_clean TEXT` field capped in the M2.4 DDL).
- **`paper_id` on `FullTextExtract`** is the stable canonical ID (e.g., `paper:doi:10.xxxx/yyy` or `paper:arxiv:2310.12345` per the project's `build_stable_id` convention from `apps/miroflow-agent/src/data_agents/normalization.py`). Callers set it; M2.3 just threads it through. The plan does NOT re-derive paper_id from a `ResolvedPaper` in this milestone ‚Äî caller (M2.4) owns that decision. For Unit tests, pass an explicit `paper_id` string.
- **Never raise on external flakiness.** HTTP 404 (arxiv withdraw), 429 (rate limit backlash), 5xx, pdfminer errors (corrupted PDF), empty-text PDFs ‚Üí all log warning, return `FullTextExtract(source="failed", fetch_error=<short explanation>)`.
- **pdfminer can take `BytesIO` or a path.** We hold bytes in memory, so `BytesIO`. No tmp files on disk, no cleanup needed. Small PDFs (<20MB) fit in memory; 4-page extraction doesn't need the full file anyway but we hash the full file for dedup.
- **PDF size cap 30MB.** If the Content-Length header (or observed stream) exceeds this, abort download and mark failed. Defensive ‚Äî arxiv papers rarely exceed 10MB.
- **Single try, no retries.** Match M2.2's "fall through on failure" pattern. M2.4 can re-queue at the batch level with different pacing.

## Open Questions

### Resolved During Planning

- **Q: Sync or async?** ‚Üí Sync. Matches M2.2 and M0.1. M2.4 can thread-pool.
- **Q: Share `_RateLimitGate` with M2.2 or duplicate?** ‚Üí Duplicate. Shared-refactor touches M2.2, which invites Codex Shape 3.
- **Q: Fetch publisher PDFs when arxiv misses?** ‚Üí No. Deferred to a future milestone. Paywalled fetching needs credentials; academic ethics + legal caution apply.
- **Q: OCR for image-only PDFs?** ‚Üí No. Return `source="failed"` with `fetch_error="pdf_text_empty"`.
- **Q: `paper_id` derivation?** ‚Üí Caller's responsibility (M2.4). M2.3 is a pure extractor.
- **Q: Abstract fallback when PDF fails?** ‚Üí Yes, if `paper.abstract` is non-empty, use it. `source="openalex"` signals semantic origin.
- **Q: Size cap?** ‚Üí 30MB upper bound on downloads. Defensive.
- **Q: What if abstract regex finds nothing but Introduction matches?** ‚Üí `abstract=None`, `intro=<text>`. We record what we can.
- **Q: What if both Abstract and Introduction regexes miss?** ‚Üí `abstract=None`, `intro=None`, `source="arxiv"` (we did download the PDF). Not a failure per se; M2.4 can decide to surface via `pipeline_issue`.

### Deferred to Implementation

- **Exact regex tuning** for section headings ‚Äî start with the patterns above; extend as test failures against real-world PDF fixtures surface edge cases.
- **Exact error-message format** for `fetch_error` ‚Äî short canonical strings per failure mode (e.g., `"http_404"`, `"pdf_parse_error"`, `"no_arxiv_id"`, `"pdf_too_large"`). Pin in tests once implementation exists.
- **pdfminer.six `laparams`** ‚Äî start default; tune if real PDFs produce unreadable output. Test fixtures stay hermetic so tuning is cheap.
- **Handling arxiv v-number in `pdf_url` when input `arxiv_id` is bare** ‚Äî arxiv accepts both `https://arxiv.org/pdf/2310.12345.pdf` and `https://arxiv.org/pdf/2310.12345v3.pdf`. Implementer picks the bare form (no version) for caching idempotence.

- [ ] **Unit 2: PDF download + sha256 + pdfminer extraction**

**Goal:** Wire HTTP download with size cap and sha256, then pdfminer-based text extraction. Expose both as module-private helpers that Unit 3 orchestrates.

**Requirements:** R4, R6, R7, R10

**Dependencies:** Unit 1.

**Files:**
- Modify: `apps/miroflow-agent/src/data_agents/paper/full_text_fetcher.py` (add `_RateLimitGate`, `_ARXIV_PDF_GATE`, `_download_pdf`, `_extract_text_from_pdf_bytes`, `_DEFAULT_TIMEOUT`, `_MAX_PDF_BYTES`).
- Test: same test file, appended.

**Approach:**
- `_RateLimitGate` duplicated from `paper/title_resolver.py` (per decision). Add a `# TODO(M0.3-ish): factor to shared module when third caller appears.` comment.
- `_ARXIV_PDF_GATE = _RateLimitGate(3.0)`.
- `_DEFAULT_TIMEOUT = 60.0` (PDFs are bigger than JSON; 60s is defensive for slow arxiv mirrors).
- `_MAX_PDF_BYTES = 30 * 1024 * 1024` = 30MB.
- `_download_pdf(url, *, http_client) -> tuple[bytes, str]`:
  - Calls `_ARXIV_PDF_GATE.wait()`.
  - `http_client.get(url, timeout=_DEFAULT_TIMEOUT, follow_redirects=True)`.
  - `response.raise_for_status()`.
  - If `Content-Length` header present and > `_MAX_PDF_BYTES` ‚Üí raise a module-private `_OversizeError`.
  - `pdf_bytes = response.content` (single read).
  - If `len(pdf_bytes) > _MAX_PDF_BYTES` (defense-in-depth if Content-Length was missing) ‚Üí raise `_OversizeError`.
  - Return `(pdf_bytes, hashlib.sha256(pdf_bytes).hexdigest())`.
- `_extract_text_from_pdf_bytes(pdf_bytes: bytes) -> str`:
  - Wraps pdfminer's `high_level.extract_text(BytesIO(pdf_bytes), maxpages=4)`.
  - Catches pdfminer exceptions, re-raises as module-private `_PdfParseError`.
  - Returns the extracted string. Empty string is valid (caller treats as "unreadable PDF").
- Owned httpx.Client: internal helper `_make_http_client()` returns `httpx.Client(timeout=_DEFAULT_TIMEOUT, trust_env=False, follow_redirects=True)`.

**Execution note:** Test-first. Use in-memory synthetic PDF bytes for pdfminer happy-path; use fake bytes like `b"%PDF-1.4 ..."` minimal valid or full generate via reportlab if installed. If neither works, use a checked-in tiny sample PDF fixture.

**Patterns to follow:**
- `paper/title_resolver.py::_RateLimitGate` (duplicate verbatim).
- `paper/title_resolver.py` ‚Äî `trust_env=False` on owned client.
- `providers/rerank.py` ‚Äî sync httpx.Client ownership pattern.

**Test scenarios:**

*Download happy paths:*
- Happy path ‚Äî Mock httpx returns 200 + PDF bytes ‚Üí `_download_pdf` returns `(bytes, sha256_hex)`.
- Happy path ‚Äî sha256 is deterministic (hashing known bytes twice yields identical hex).
- Happy path ‚Äî URL passed to `http_client.get` is exactly `https://arxiv.org/pdf/<id>.pdf` (no version, follows arxiv convention).

*Download edge cases:*
- Edge case ‚Äî `Content-Length: 50000000` (50MB) ‚Üí raises `_OversizeError`, never reads body (verify via spy on response).
- Edge case ‚Äî No Content-Length header but actual body > 30MB ‚Üí raises `_OversizeError` post-hoc.
- Edge case ‚Äî `Content-Length: 1024` ‚Üí small PDF, succeeds.

*Download error paths:*
- Error path ‚Äî HTTP 404 ‚Üí `response.raise_for_status()` raises `httpx.HTTPStatusError` (propagates to orchestrator which converts to fetch_error).

- Error path ‚Äî `httpx.ConnectError` ‚Üí propagates.

- Integration ‚Äî Two consecutive `_download_pdf` calls: second observes at least 3.0s gap (patched clock or injected gate).

- Edge case ‚Äî Non-PDF bytes (random junk) ‚Üí raises `_PdfParseError`.
- Edge case ‚Äî Valid PDF but image-only (no extractable text) ‚Üí returns empty string `""` (NOT a raise; caller treats as missing).
- Edge case ‚Äî Valid PDF with 10 pages ‚Üí `maxpages=4` cap means returned text only covers first 4 pages.

*Client ownership:*
- Contract ‚Äî `_make_http_client()` returns an `httpx.Client` with `trust_env=False` and `follow_redirects=True`.

**Approach:**
- Public `fetch_and_extract_full_text(paper: ResolvedPaper, *, paper_id: str, http_client: httpx.Client | None = None) -> FullTextExtract`.
- If `http_client is None`, use `_make_http_client()`. Track ownership; close if owned and no caller-managed context.
- If `paper.arxiv_id` truthy:
  - Construct `pdf_url = f"https://arxiv.org/pdf/{paper.arxiv_id}.pdf"`.
  - Try `_download_pdf(pdf_url, http_client=...)` + `_extract_text_from_pdf_bytes(bytes)` + `_split_abstract_intro(text)`.
  - On success ‚Üí `FullTextExtract(source="arxiv", fetch_error=None, ...)`.
  - On any exception (`httpx.HTTPStatusError`, `_OversizeError`, `_PdfParseError`, `httpx.RequestError`), log warning, `fetch_error = <canonical_tag>`, fall through.
- If PDF path failed AND `paper.abstract` is non-empty ‚Üí `FullTextExtract(abstract=paper.abstract, intro=None, source="openalex", fetch_error=None, pdf_url=paper.pdf_url, pdf_sha256=None, paper_id=paper_id)`.
- Else ‚Üí `FullTextExtract(source="failed", fetch_error=<last_error_tag or "no_arxiv_id">, ...)`.
- Canonical `fetch_error` tags: `"http_404"`, `"http_429"`, `"http_5xx"`, `"timeout"`, `"network"`, `"pdf_too_large"`, `"pdf_parse_error"`, `"pdf_empty_text"`, `"no_arxiv_id"`.
- Never raise from this function on external flakiness. Raises only on argument-type errors (caller bug).

**Execution note:** Test-first. Patch `_download_pdf` + `_extract_text_from_pdf_bytes` + `_split_abstract_intro` independently so orchestrator tests isolate cascade behavior.

**Patterns to follow:**
- `paper/title_resolver.py::resolve_paper_by_title` cascade ‚Äî same short-circuit-on-success, never-raise-on-failure shape.

**Test scenarios:**

*Happy paths:*
- Happy path ‚Äî `paper.arxiv_id = "2310.12345"`, download succeeds, extraction succeeds, split returns both abstract+intro ‚Üí `source="arxiv"`, `pdf_sha256` populated, `fetch_error=None`.
- Happy path ‚Äî arxiv path returns PDF but split finds no Abstract heading ‚Üí `abstract=None`, `intro=<text>`, `source="arxiv"`, `fetch_error=None` (PDF was still retrieved).
- Happy path ‚Äî arxiv path returns empty text (image-only PDF) ‚Üí `abstract=None`, `intro=None`, `source="arxiv"`, `fetch_error="pdf_empty_text"` (OR `source="failed"` ‚Äî pin implementer choice). Recommended: `source="failed"` because we got no usable content.
- Happy path ‚Äî No arxiv_id, `paper.abstract = "We study ..."` ‚Üí `source="openalex"`, `abstract="We study ..."`, `intro=None`, `pdf_sha256=None`.

*Cascade:*
- Integration ‚Äî arxiv download fails 404, `paper.abstract` set ‚Üí falls through to openalex, returns `source="openalex"`.
- Integration ‚Äî arxiv download fails 429, `paper.abstract` set ‚Üí `source="openalex"`.
- Integration ‚Äî arxiv download succeeds but pdfminer raises ‚Üí falls through to openalex if `paper.abstract` set, otherwise `source="failed"`, `fetch_error="pdf_parse_error"`.

*Total-fail paths:*
- Error path ‚Äî No arxiv_id, no abstract ‚Üí `source="failed"`, `fetch_error="no_arxiv_id"`.
- Error path ‚Äî arxiv 404 + no abstract ‚Üí `source="failed"`, `fetch_error="http_404"`.
- Error path ‚Äî arxiv succeeds, pdfminer fails, no abstract ‚Üí `source="failed"`, `fetch_error="pdf_parse_error"`.
- Error path ‚Äî Oversize PDF + no abstract ‚Üí `source="failed"`, `fetch_error="pdf_too_large"`.

*Input validation:*
- Contract ‚Äî `paper` is a `ResolvedPaper`; passing a dict or `None` raises `TypeError` / `AttributeError`. (Programmer error.)
- Contract ‚Äî `paper_id` required; passing `None` raises `TypeError`.

*Client ownership:*
- Contract ‚Äî Function creates its own `httpx.Client` with `trust_env=False` when `http_client` is None. Patch `src.data_agents.paper.full_text_fetcher.httpx.Client` (use the real-class-capture idiom).
- Contract ‚Äî Caller-injected `http_client` is NOT closed by the function.

## Risks & Dependencies

| Risk | Mitigation |
|------|------------|
| pdfminer.six hangs on pathological PDFs (rare but documented) | httpx timeout limits download; pdfminer itself has no timeout but `maxpages=4` bounds work. Accept this; if it surfaces in production, wrap in `signal.alarm` or subprocess later. |
| arxiv rate-limit backlash (429) despite 3s gate | Cascade falls through to `source="openalex"` when possible, else `source="failed"`. M2.4 batch-level retry policy handles aggregate. |
| Very old arxiv papers return scanned/image PDFs (no extractable text) | `_extract_text_from_pdf_bytes` returns empty string, caller treats as empty ‚Üí `fetch_error="pdf_empty_text"`, `source="failed"`. Document this expectation. |
| Codex duplicates `_RateLimitGate` poorly (e.g., subtly different lock semantics from M2.2's) | Explicit brief: copy M2.2's `_RateLimitGate` class verbatim. Cross-validate with a `diff` between the two class definitions before ship. |
| Codex adds pdfminer timeout logic we didn't plan for | Not in requirements. Brief explicitly rejects scope creep beyond R1-R11. |
| Codex touches `paper/title_resolver.py` to share `_RateLimitGate` (Shape 3 from memory/feedback_codex_deviations.md) | Brief explicitly forbids. Cross-validation `git diff --stat` catches. Same anti-drift mitigation that worked in M2.2. |
| Codex uses pypdf or pdfplumber instead of pdfminer.six | Brief specifies pdfminer.six, cites its location in deps. |
| 30MB cap too low for some books-as-arxiv PDFs | Adjustable constant; bump to 60MB if real data surfaces the need. Start conservative. |
| Abstract/intro regex misses common LaTeX `\section{Introduction}` rendering (rendered as "Introduction" by pdfminer but preceded by section-number artifacts like "I." or Chinese numerals) | Test fixtures will cover observed patterns. Unit 1 split tests capture 10+ variations; extend iteratively. Accept 70-80% first-pass hit rate; M4 retrieval chunking doesn't strictly need both fields. |
| Tests require a real PDF fixture which is binary and bloats the repo | Option A: generate minimal PDF via `reportlab` at test-setup (if reportlab is a dep or easy to add). Option B: commit a tiny hand-crafted 2-3KB PDF. Option C: patch `pdfminer.high_level.extract_text` entirely and feed it canned strings ‚Äî most hermetic. Recommend Option C for most tests, Option B for one smoke test. Implementer decides at RED time. |
